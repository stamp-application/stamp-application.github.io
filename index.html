Store the version number in arbitrary number of different files.

about

Stamp is a command line application that makes versioning just a little bit easier. 

$ stamp major:up
$ stamp minor:up
$ stamp patch:up

The role of each of the above commands is:

  • to guess the current version number of your project,
  • to increase the version number,
  • store the version in arbitrary files,
  • finally tag your repo with $ git tag command.

For a detailed technical documentation please refer to README.md.

examples

#1 / puppet module

The project contains two text files: metadata.json and stamp.yml. 

filename: metadata.json

{
  "name": "gajdaw-php_phars",
  "version": "0.1.4",
  "author": "gajdaw"
}

filename: stamp.yml

filename:    'metadata.json'
regex:       '/"version": "(?P<version>[\d\.]+)",/'
replacement: '"version": "{{ version }}",'

The command: 

$ stamp minor:up

would store in metadata.json file version 0.2.0: 

filename: metadata.json

{
  "name": "gajdaw-php_phars",
  "version": "0.2.0",
  "author": "gajdaw"
}

The same version number would also be tagged with:

$ git tag -a v0.2.0 -m "Release 0.2.0"

The above files come from my puppet module puppet-php_phars.

#2 / html/css project

The project contains two text files: version.txt and stamp.yml. 

filename: version.txt

1.34.156

filename: stamp.yml

filename:    'version.txt'
regex:       '|(?P<version>[\d\.]+)|'
replacement: '{{ version }}'

The command: 

$ stamp patch:up

would store in version.txt file version 1.34.157: 

filename: version.txt

1.34.157

The same version number would be also tagged with:

$ git tag -a v1.34.157 -m "Release 1.34.157"

The above files come from this very project: stamp web site.
